FishTones.js: built for real life

fishTones.js library has been originally developed to sustain research applications in proteomics at Genentech. Building tools to analyze complex mass spectrometry data was the drive to desgign the features presented across these pages.

However, we have not made here an exhaustive tour of all fishTones.js features. More are demonstrated in the unit test page (see github README). Among them are for example widgets to display multiple XIC tracks, interactively measure XIC peak AUC ratios, some basic statistics and maths functions and much more.

Altering rendering

Embedding widgets into different context often mean to adapt the rendering. Either by adapting the style to a particular web application or exporting images to presentations or papers.

CSS edition

fishTones.js widgets are designed to be included in web application. To adapt any rendering aspect (color, font, line width etc.), the easiest way is to create a css file and import it after css/fishtones-js.css. For mere mortals, css is more a matter of trial and errors, so Chrome developer tools or firebug will be your best friend!

Exporting to SVG

As soon as a graph is rendered on a screen a scientist will come and ask how to export it for a paper. A raster image might not fit many journals preferring vector format. Fortunately enough, widgets are built in SVG and can be therefore imported into vector graphic editor tools such as Inkscape or Adobe Illustrator (it's sad to say so, but Illustrator looks to handle better some fancy SVG exports here).

The easiest way to export SVG is to use Mike Bostock (the creator of d3.js) svg crowbar. Simply add the link on your bookmark tool bar, and click on it when you are on a web page. Save buttons will appear over each rendered widget on the page.

More applications

Multiple applications have been internally developed to support research scientists at Genentech, based on fishTones.js. Here are snapshots from a several of them, to illustrate how basic components can be assemble together.

Protein match

All the PSM for a protein across 4 runs are displayed via the open source pViz.js library. Identified peptides are plot as blue lines, and red circles stand for oxidation. It is possible to zoom in on the protein sequence, to click on peptide in order to review the annotated peptide or plot the targeted XICs.

XIC across runs

Reviewing XIC for a given peptide across a series can be achieved. In this snapshot, we demonstrate the use of an internal retention time algorithm. Having the retention times aligned allow to one LC time range to be selected across all the runs and has proven efficient to measure multiple peptide abundance at once.

PSM elucidation

Elucidating a peptide sequence (and modifications) is a tedious and repetitive task for mass spectrometrist. We developed a tool to facilitate this task, highlighting fragment distance with a know theoretical value (loss, modification, amino acid), linked to the edition of the peptide sequence.

Fishing PSM tags

Instead of identifying full peptides, a backend application can search for smaller peptide tags within one or more MS/MS runs. Peptide alignment are matched and spectra are aligned so that they all zoom in the same area.

Spectra network

This graph represents spectra (gray circles) matching to each others and to peptides (orange circle). Red edges therefore represent PSM and black one SSM (the bolder ones indicate non trivial mass shifts). Vertices can be clicked to display details and edges to show the alignments.
